# Antigravity Practice

This directory is for practicing and testing with the Antigravity agent.

## Antigravityを用いた開発のベストプラクティス

Antigravity（AIエージェント）を最大限に活用するための推奨されるアプローチです。

### 1. 指示を具体的にする

曖昧な要求を避け、達成したい具体的なゴールを伝えてください。

- **Bad**: 「コードを改善して」
- **Good**: 「この関数のメモリ消費を抑えるために、イテレータを使用するようにリファクタリングして」

### 2. コンテキストを提供する

作業対象のファイルを開いた状態にしたり、関連するファイルパスを明示的に伝えることで、AIが現在の状況を正確に把握できるようになります。

### 3. 反復的なプロセス（PLANNING → EXECUTION → VERIFICATION）

Antigravityは自律的なタスク遂行が可能です。

- **PLANNING**: 複雑なタスクではまず計画を立てさせ、内容を確認・承認してください。
- **EXECUTION**: 承認された計画に基づき、AIがコードを生成・変更します。
- **VERIFICATION**: 変更が正しく機能するか、テストやブラウザによる確認を依頼してください。

### 4. ツールをフル活用させる

Antigravityはファイル操作、ターミナルコマンドの実行、ウェブ検索、ブラウザ操作が可能です。
「ブラウザで表示を確認して」「テストを実行して結果を教えて」といった指示を組み合わせることで、開発効率が向上します。

### 5. フィードバックを恐れない

AIの提案が意図と異なる場合は、遠慮なく指摘してください。誤解を正すことで、より精度の高い成果物を得ることができます。

## 高度なトピックと関連エコシステム

Antigravityを取り巻くツールや概念についての解説です。

### Kiroとの違い：Swarm vs Architect

AntigravityとKiroはどちらもエージェント型IDEですが、その設計思想に違いがあります。

- **Antigravity (Swarm)**: 複数のサーフェス（エディタ、ターミナル、ブラウザ）を自在に操り、自律的にタスクを完遂させる「働き手」としての側面が強いです。
- **Kiro (Architect)**: 仕様（Spec）駆動開発に重きを置き、厳格な構造とテストによる検証を重視する「設計者」としての側面が強いです。

### Claude Codeとの比較

Anthropicが提供する **Claude Code** は、ターミナルネイティブなエージェントです。AntigravityはIDEベースで視覚的なタスク管理（Task View）が可能ですが、Claude CodeはCLIから迅速にコードベースを操作することに特化しています。どちらも後述するMCPをサポートしています。

### MCP (Model Context Protocol)

**MCP** は、AIモデルをローカルファイル、データベース、APIなどの外部データソースやツールに接続するための標準規格です。AntigravityやClaude CodeはMCPを利用して、エージェントがアクセスできる「スキル」や「データ」を動的に拡張できます。

### Skill Agent (Agent Skill)

**Skill Agent**（またはAgent Skill）は、特定のドメイン知識やワークフローをパッケージ化したものです。プロジェクト内に `SKILL.md`（または `.agent/skills/`）を配置することで、エージェントに「新しい能力」を覚えさせることができます。
